# CLAUDE.md

## Design Context

### Users
- **目标用户**: AI 工程师、全栈开发者、技术负责人
- **使用场景**: 浏览、搜索和发现 AI Skills 生态中的可用技能，了解趋势和热门工具，管理自己的技能配置
- **核心任务**: 快速找到所需技能、了解生态动态、做出技术选型决策

### Brand Personality
- **直观** — 信息层级清晰，操作路径短，用户无需学习即可上手
- **极客** — 终端/CLI 美学，等宽字体标签，开发者文化认同感
- **前沿** — 紧跟 AI 工程生态最新动态，呈现技术前沿趋势

### Aesthetic Direction
- **视觉基调**: 极简、技术感、暗色优先的终端美学
- **暗色模式**: 深黑底色 `#09090b`，表面色 `#0a0a0a`，边框 `#333`
- **亮色模式**: 浅灰底色 `#eeeef0`，作为可选替代
- **字体**: Inter（正文/UI） + JetBrains Mono（数据/标签/代码）
- **图标**: Material Icons Outlined，线性风格
- **品牌标识**: 白底黑字 "S" 方块 + "SKILLS.SH" 粗体标题
- **色彩调色板**:
  - 主色：蓝色渐变 `blue-500 → blue-400`（下载量、链接高亮）
  - 辅助色：绿色（Trending 状态、CLI 命令）、橙色（Hot 状态）、紫色（组织分布、更新命令）、粉色/青色/黄色（图表辅助色）
  - 状态色：`green-400`（LIVE 指示器）
- **情感目标**: 探索感 — 让用户感受到发现新工具的兴奋和好奇心
- **反模式**: 避免过度装饰、避免圆润可爱风格、避免大面积鲜艳色块

### Design Principles
1. **信息密度优先** — 在有限空间内展示尽可能多的有用数据，减少装饰性留白
2. **终端即美学** — 等宽字体、大写标签、代码块风格是设计语言的核心，不是妥协
3. **渐进式发现** — 通过 hover 效果、颜色变化和交互反馈引导用户深入探索
4. **一致的克制** — 颜色仅用于传达语义（状态、分类、交互），不用于纯装饰
5. **开发者共鸣** — 设计决策应让开发者感到"这是为我打造的"，而非面向大众消费者

### Technical Stack
- **框架**: Next.js 16 (App Router) + React 19
- **样式**: Tailwind CSS v4 (`@tailwindcss/postcss`)
- **语言**: TypeScript
- **字体加载**: `next/font/google` (Inter, JetBrains Mono)
- **图标**: Google Material Icons Outlined (CDN)
- **工具库**: clsx (条件类名), cheerio (数据抓取)
- **视频生成**: Remotion (宣传视频)
- **开发工具**: Agentation (开发模式可视化反馈)
- **部署**: Vercel
